# Opal Slice 2: Calculator — Numbers & Functions

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add function definitions, user-defined function calls with recursion, `return` statements, and type annotations (parsed, not enforced) — enough to run `factorial(10)` and `fibonacci(35)`.

**Architecture:** Extends the Slice 1 pipeline (lexer → parser → interpreter). Adds `StmtKind::FuncDef` and `StmtKind::Return` to the AST, `Value::Function` to the runtime, and function call/return handling to the interpreter. Type annotations are parsed into the AST but ignored at runtime (Phase 1 is dynamically typed). The existing `Environment` with `push_scope`/`pop_scope` provides lexical scoping for function call frames.

**Tech Stack:** Same as Slice 1 — Rust, `logos`, `clap`, `criterion`, `proptest`, `miette`, `thiserror`.

**Design doc:** `docs/plans/2026-03-03-production-implementation-design.md`

**Spec reference:** `Opal.md` lines 142-146 (function_def, params BNF)

### Task 2: Parse function definitions

**Files:**

**What to add:**

Add a `parse_function_def` method to `Parser`:
```rust
fn parse_function_def(&mut self) -> Result<Stmt, ParseError> {
    let start = self.current_span();
    self.advance(); // consume 'def'

    let name = self.expect_identifier()?;
    self.expect_token(&Token::LParen, "(")?;
    let params = self.parse_params()?;
    self.expect_token(&Token::RParen, ")")?;

    // Optional return type: -> Type
    let return_type = if self.check(&Token::Arrow) {
        self.advance();
        Some(self.expect_identifier()?)
    } else {
        None
    };

    self.expect_newline()?;
    let body = self.parse_block()?;
    self.expect_token(&Token::End, "end")?;
    let end = self.previous_span().end;

    Ok(Stmt {
        kind: StmtKind::FuncDef {
            name,
            params,
            return_type,
            body,
        },
        span: Span {
            start: start.start,
            end,
        },
    })
}
```

Add a `parse_params` method:
```rust
fn parse_params(&mut self) -> Result<Vec<Param>, ParseError> {
    let mut params = Vec::new();

    if self.check(&Token::RParen) {
        return Ok(params);
    }

    loop {
        let name = self.expect_identifier()?;

        // Optional type annotation: : Type
        let type_annotation = if self.check(&Token::Colon) {
            self.advance();
            Some(self.expect_identifier()?)
        } else {
            None
        };

        // Optional default value: = expr
        let default = if self.check(&Token::Eq) {
            self.advance();
            Some(self.parse_expression(0)?)
        } else {
            None
        };

        params.push(Param {
            name,
            type_annotation,
            default,
        });

        if !self.check(&Token::Comma) {
            break;
        }
        self.advance(); // consume comma
    }

    Ok(params)
}
```

**Modify `parse_statement`** to check for `def` before parsing expressions:

Also add `return` parsing:

```rust
fn parse_return_statement(&mut self, start: Span) -> Result<Stmt, ParseError> {
    self.advance(); // consume 'return'
    let value = if self.is_at_end() || self.check(&Token::Newline) || self.check(&Token::End) {
        None
    } else {
        Some(self.parse_expression(0)?)
    };
    self.expect_statement_end()?;
    let end = value.as_ref().map_or(start.end, |e| e.span.end);
    Ok(Stmt {
        kind: StmtKind::Return(value),
        span: Span {
            start: start.start,
            end,
        },
    })
}
```

### Task 3: Add Value::Function to runtime

**Files:**
- Modify: `crates/opal-runtime/src/value.rs`

**Add a `Function` variant to `Value`:**

```rust
use crate::function::UserFunction;

// In the Value enum:
/// User-defined function
Function(UserFunction),
```

**Create `crates/opal-runtime/src/function.rs`:**
```rust
/// A user-defined function stored as a runtime value.
/// Stores the parameter names and body AST for later execution.
/// Uses String representations to avoid coupling runtime to parser types.
#[derive(Debug, Clone)]
pub struct UserFunction {
    pub name: String,
    pub params: Vec<String>,
    /// The function body is stored as an opaque pointer to the AST.
    /// The interpreter knows how to evaluate it.
    /// We use a Vec<u8> serialization... no, that's overengineered.
    /// Instead, store a reference ID into a function table.
    pub body_id: usize,
}
```

Wait — storing AST nodes in `Value` requires the runtime crate to depend on the parser crate, creating a circular dependency (parser → lexer, interp → parser + runtime, runtime → parser would be circular).

**Better approach:** Store function bodies in a side table in the Interpreter, and store only a `FunctionId(usize)` in `Value`. This keeps the runtime crate decoupled from the parser.

```rust
// In value.rs — add to Value enum:
/// User-defined function (ID into the interpreter's function table)
Function(FunctionId),
```

**Create `crates/opal-runtime/src/function.rs`:**
```rust
/// An opaque identifier for a user-defined function stored in the interpreter
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct FunctionId(pub usize);
```

**Update `crates/opal-runtime/src/value.rs`** Display for Function:
```rust

```

**Update `crates/opal-runtime/src/lib.rs`:**
```rust

// ...

```

**Handle `StmtKind::FuncDef` in `eval_stmt`:**
```rust
StmtKind::FuncDef { name, params, body, .. } => {
    let id = FunctionId(self.functions.len());
    self.functions.push(StoredFunction {
        name: name.clone(),
        params: params.iter().map(|p| p.name.clone()).collect(),
        body: body.clone(),
    });
    self.env.set(name.clone(), Value::Function(id));
}
```

**Handle `StmtKind::Return`** — use a special error type to propagate returns:

Add a new variant to `EvalError`:
```rust
#[error("return")]
Return(Value),
```

In `eval_stmt`:
```rust
StmtKind::Return(expr) => {
    let val = match expr {
        Some(e) => self.eval_expr(e)?,
        None => Value::Null,
    };
    return Err(EvalError::Return(val));
}
```

**Update function call handling** in `eval_expr` for `ExprKind::Call`:

After the builtin check, before the `Err(UndefinedVariable)`:
```rust
// Try user-defined functions
if let Some(value) = self.env.get(&func_name).cloned() {
    if let Value::Function(id) = value {
        let stored = &self.functions[id.0].clone();
        if arg_values.len() != stored.params.len() {
            return Err(EvalError::TypeError(format!(
                "{}() expected {} arguments, got {}",
                func_name,
                stored.params.len(),
                arg_values.len()
            )));
        }
        // Create new scope for function call
        self.env.push_scope();
        for (param_name, arg_val) in stored.params.iter().zip(arg_values) {
            self.env.set(param_name.clone(), arg_val);
        }
        let result = self.eval_block(&stored.body);
        self.env.pop_scope();
        return match result {
            Ok(val) => Ok(val),
            Err(EvalError::Return(val)) => Ok(val),
            Err(e) => Err(e),
        };
    }
}
```
