# Opal Slice 4: OOP — Classes & Modules

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add classes with `needs` (constructor params), instance variables (`.field`), methods, modules, `.new()` constructors, and `from X import Y` — enough to run the Shapes target program.

**Architecture:** Classes are stored as `Value::Class(ClassId)` referencing a `StoredClass` with needs declarations, methods, and module path. Instances are `Value::Instance(InstanceId)` referencing a `StoredInstance` with class ID and field map. Modules are `Value::Module(ModuleId)` referencing a namespace of bindings. Method calls on instances dispatch to the class's method table. `.new()` is a built-in class method that validates named args against `needs` declarations. Instance variables use `.field` syntax which resolves against the current `self` instance. `from X import Y` copies bindings from a module's namespace.

**Tech Stack:** Same as Slices 1-3.

**Spec reference:** `Opal.md` lines 163-166 (class/module BNF), 213 (needs), 258-266 (imports)

## Context

**Target program:**
```opal
module Shapes
  class Circle
    needs radius: Float

    def area() -> Float
      Math.pi() * .radius ** 2
    end
  end

  class Rectangle
    needs width: Float
    needs height: Float

    def area() -> Float
      .width * .height
    end
  end
end

from Shapes import Circle, Rectangle

shapes = [Circle.new(radius: 5.0), Rectangle.new(width: 3.0, height: 4.0)]
for shape in shapes
  print(f"Area: {shape.area()}")
end
```

**Scoped out:** Protocols, visibility modifiers, inheritance, `self` keyword (implicit via `.field`). `Math.pi()` simplified to a stdlib function.

### Task 4: Implement classes, instances, modules, and imports in interpreter

**Files:**
- Modify: `crates/opal-interp/src/eval.rs`

**Stored types:**
```rust
struct StoredClass { name: String, needs: Vec<(String, Option<String>)>, methods: Vec<StoredFunction> }
struct StoredInstance { class_id: ClassId, fields: HashMap<String, Value> }
struct StoredModule { name: String, bindings: HashMap<String, Value> }
```

**Key behaviors:**
1. `ClassDef` → store class, bind name to `Value::Class(id)`
2. `ModuleDef` → evaluate body in a scope, capture bindings as module
3. `FromImport` → look up module, copy named bindings into current scope
4. `Class.new(field: val)` → validate against needs, create instance with fields
5. `instance.method()` → look up method in class, execute with `.field` access
6. `ExprKind::InstanceVar` → look up field on current `self` instance
7. Instance method calls push a scope with `self` bound, `.field` resolves from `self`
8. Add `Math.pi()` to stdlib as a simple builtin module function

**Tests:** class instantiation, method calls, modules, from import, the full target program.

**Step 1:** Implement everything. Run `cargo test --package opal-interp`.
