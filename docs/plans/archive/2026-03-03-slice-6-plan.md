# Opal Slice 6: Actors (Synchronous)

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add actors with `receive`/`case` message handlers, `.send()`, `reply`, symbol literals, instance variable assignment, and `await` — synchronous actors first, async/tokio deferred.

**Architecture:** Actors are stored as special instances with a receive handler (list of match cases). `.send(:msg)` dispatches to the receive handler synchronously. `reply` sets a return value captured by `send`. `await` is a no-op in synchronous mode. Symbols (`:name`) become `Value::Symbol`. Instance variable assignment (`.field = expr`) updates the actor/instance's fields.

**Simplification from design doc:** The target program uses `.count += 1` — we'll use `.count = .count + 1` instead, avoiding compound assignment operators.

---

## Target program (adapted):
```opal
actor Counter
  def init()
    .count = 0
  end

  receive
    case :increment
      .count = .count + 1
    case :get
      reply .count
  end
end

counter = Counter.new()
counter.send(:increment)
counter.send(:increment)
print(await counter.send(:get))
```
Expected output: `2`

**Parser:** Symbol in `parse_primary` for `Token::Symbol`. Await as unary prefix. Actor definition. Reply statement. Instance variable assignment when `.field` is followed by `=`.
