# Opal Slice 5: Error Handling — Result, Match, Requires

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add `match`/`case` pattern matching, `Result` and `Option` types via enum-like constructors (`Ok`, `Error`, `Some`, `None`), `requires` preconditions, `raise`/`try`/`catch`/`ensure` — enough to run the error handling target program.

**Architecture:** `match`/`case` is a new expression that tests values against patterns. `Ok(v)` and `Error(v)` are constructor calls that produce tagged wrapper values (`Value::Ok`/`Value::Error`). `requires` is a statement that raises on failure. `try`/`catch`/`ensure` uses a new `EvalError::Raise` variant. Pattern matching destructures tagged values and literals.

## Target program:
```opal
def divide(a: Float, b: Float) -> Result[Float, String]
  requires b != 0.0, "division by zero"
  Ok(a / b)
end

match divide(10.0, 3.0)
  case Ok(result)
    print(f"Result: {result}")
  case Error(msg)
    print(f"Error: {msg}")
end
```

### Task 1: Add match, require, try/catch, and Result/Option AST nodes

**Files:** Modify `crates/opal-parser/src/ast.rs`

**Add to `ExprKind`:**
```rust
/// Match expression
Match { subject: Box<Expr>, cases: Vec<MatchCase> },
```

**Add to `StmtKind`:**
```rust
/// Requires precondition
Requires { condition: Expr, message: Option<Expr> },
/// Try/catch/ensure
TryCatch { body: Vec<Stmt>, catches: Vec<CatchClause>, ensure: Option<Vec<Stmt>> },
/// Raise an error
Raise(Expr),
```

**Add structs:**
```rust
#[derive(Debug, Clone)]
pub struct MatchCase {
    pub pattern: Pattern,
    pub body: Vec<Stmt>,
}

#[derive(Debug, Clone)]
pub enum Pattern {
    Identifier(String),        // binds variable
    Literal(Expr),             // matches literal value
    Constructor(String, Vec<Pattern>),  // Ok(x), Error(msg), Some(v)
    Wildcard,                  // _
}

#[derive(Debug, Clone)]
pub struct CatchClause {
    pub error_type: Option<String>,
    pub var_name: Option<String>,
    pub body: Vec<Stmt>,
}
```

### Task 4: Implement match, requires, try/catch, raise in interpreter

**Files:** Modify `crates/opal-interp/src/eval.rs`

Key behaviors:
1. `Ok(expr)` / `Error(expr)` / `Some(expr)` — builtin constructor functions
2. `match` — evaluate subject, try each case pattern, execute matching body
3. Pattern matching: `Ok(x)` destructures `Value::Ok`, binding inner value to `x`
4. `requires cond, msg` — if cond is false, raise RuntimeError
5. `raise expr` — emit `EvalError::Raise(value)`
6. `try/catch/ensure` — run body, catch Raise errors, always run ensure
